# 開発の外へ：AIと進める、どんな意思決定にも

英語版：[../beyond-code.md](../beyond-code.md)

コードレビューは Qorum の一番わかりやすい適用先であって、境界ではありません。PRには構造が最初から付いてくる——差分という具体物、開く→レビュー→merge というパイプライン、「どれだけ・どこを変えたか」という後戻りやすさの目印。一方、AIと進める多くの意思決定——戦略の場、採用、価格——には、その構造がありません。Qorum は、同じ構造をそこに貸す方法です。

## 写像

| コードでは | どんな意思決定でも |
|---|---|
| 差分 | 検討している提案／AIの出力 |
| 実装役 | 案を作るAI（または自分） |
| レビュー役 | 改善でなく、粗を探させる別の主体 |
| リスク階層 | どれだけ後戻りでき、外したときどれだけ高くつくか |
| merge | 結論を採用し、動くこと |
| 保護対象（protected paths） | 不可逆な領域——お金・人・ブランド・法務 |
| 人間の承認役 | 高リスクの判断だけを引き受けるあなた |

動かせない原則は同じ。**レビュー役は低リスクを通してよい。中リスクは人間が一度目を通し、不可逆なものは人間だけが裁可する。**

## 意思決定のための3段階

- 🟢 **低——AIに流させる。** 文言、論点の整理、比較表、調査メモ。一方のAIが案を出し、もう一方が批判し、あなたは結果を受け取る。外したときのコスト：やり直すだけ
- 🟡 **中——レビューの後、人間が一目見る。** 施策、優先順位、顧客向けの草案、採用要件、今週の計画。AIの批判が先に走り、それから自分が見るか、一晩置いてから決める
- 🔴 **高——AIは助言、人間が決める。** 事業方針、価格、採用と解雇、法務・財務・ブランドの取り返しのつかないもの。AIはあらゆる側を論じてよい。でも、決めはしない

## なぜ、その都度の判断に任せないのか

失敗の仕方が決まっているからです。速くて流暢なAIを相手にすると、**低リスクの判断と不可逆の判断が、同じに感じられてくる**——どちらも自信たっぷりの段落で届くから。階層は、価格の判断を文言修正と同じ反射で扱うことを止めるためにあります。線は、夜11時にその都度ではなく、落ち着いているうちに一度引く。

これが、この方法の中心にある反転です。役に立つ問いは「AIに何を渡せるか？」ではなく、**「AIに何を決めさせないと決め、それを声に出して言ったか？」**

## 正直な但し書き

コードのゲートは、何度も回して鍛えた部分です。この一般化は、同じ"形"を広い的に向けたもの——構造は確かですが、表をそのまま輸入しないでください。あなたの仕事で本当に後戻りできるものから、自分の階層を引く。要は表の中身ではなく、問いの方です。

> あなたは今、AIに、どこまで「決めさせて」いますか。ただ、自信たっぷりに答えたから、というだけの理由で。
