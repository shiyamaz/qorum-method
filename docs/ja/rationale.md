# なぜ Qorum はこの形なのか

英語版：[../rationale.md](../rationale.md)

これは、ホワイトボードで設計してから使い道を探した枠組みではありません。一人会社を「小さなAI開発組織」として数ヶ月回し、どこで実際に壊れたかを見た、その**残り物**です。

## 構成

人間ひとり。AIエージェント数体。ソロだと、誘惑は両極のどちらかに振れます。

- **自分がボトルネックになる。** 全部を自分でレビューして merge する。安全だが、それなら「レビューする」二体目のAIは芝居で、判断はまだ自分一人＝しかも疲れている。
- **機械を信用する。** AIに実装させ、AIに承認させて end-to-end。速い——ある日、自動レビューが CI 設定や秘密鍵の扱い、千行のリファクタを、にこやかに承認するまでは。

どちらも失敗します。前者はエージェントを持つ意味を捨てる。後者は、レビューの"実体"なしに"安心"だけを作る——警戒を下げるぶん、レビューが無いより危険です。

## 残って動いていたもの

現実とぶつかって生き残ったのは、狭くて退屈なものでした。だからこそ効きます。

1. **空気でなく、役割。** 書く者は、通す者ではない。両方が「自分」でも、分離は構造でなければならない。書いた本人は、自分の変更の最悪の審判だから。

2. **リスクは一様でない——だから自律も一様であってはならない。** 誤字直しとマイグレーションは同じ事象ではない。同じだけ「AIが勝手に merge してよい」を与えるのが、本当の間違い。ゲートの仕事は、自律を安いところに使い、高いところで出し惜しむこと。

3. **レビュー役の権限は、通せる範囲で縛る。** 保護対象の変更を通せるレビュー役は、レビュー役ではなく、緑チェックつきの負債。だから——安全な段は通してよく、危険な段は**通してはならない**（規約として。branch protection を入れれば、構造的に通せなくなる）。人間は、まさに——そして、そこだけ——残る。

4. **偏りは常に安全側へ。** 「迷ったら中。」不要な60秒の猶予のコストはごくわずか。誤って"低"にした秘密情報の変更のコストは、取り返しがつかないことがある。

## 正直なところ

「AIエージェントとの働き方」の多くは、ハイライト集です——勝ち、速さ、気の利いたプロンプト。このゲートは反対側から来ています：merge **すべきでなかった**のに、危うく通りかけた変更から。失敗が具体的で再現的だったから存在する。そして、夜11時に言い訳して飛ばせないルールは、忘れてしまう原則よりも価値がある。

ひとつだけ持ち帰るなら——**AIに「信用できる相手であれ」と求めるのをやめ、通してよい範囲を縛れば、信用の問いそのものが消える。**
